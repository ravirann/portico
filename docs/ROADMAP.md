# Portico Roadmap

## Phase 0 — Project foundation

- Repository structure.
- Open-source community files.
- CI, linting, tests, and release workflow.
- Basic architecture documentation.
- Branding assets.

## Phase 1 — Runtime foundation

- Tenant, target, workflow, and run model.
- Browser worker execution loop.
- Deterministic step runner.
- Run logs, screenshots, and artifacts.
- Environment-based secret injection.
- Basic retry and timeout policies.

## Phase 2 — Operator console

- Workflow registry.
- Run timeline.
- Screenshot and trace viewer.
- Failure classification.
- Manual review and escalation queue.
- Tenant-aware configuration.

## Phase 3 — AI-assisted recovery

- Page understanding helpers.
- Selector repair suggestions.
- Structured extraction validation.
- Human-in-the-loop review.
- Prompt/version tracking.
- Eval suite for recovery quality.

## Phase 4 — Production readiness

- Multi-tenant isolation hardening.
- Worker autoscaling.
- Secrets manager integration.
- Audit logs.
- Role-based access control.
- Workflow pack/plugin system.
- Deployment guides.

## Phase 5 — Ecosystem

- Public adapter SDK.
- Example workflow packs.
- Hosted demo environment.
- Community templates.
- Benchmark suite.
